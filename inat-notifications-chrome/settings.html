Notifications to fetch

Unread only (default): fetches notifications you have not yet viewed on iNaturalist. This is the normal mode.

Read only (recovery): fetches notifications that iNaturalist already considers read. This is useful when notifications you wanted to see got marked read before you had a chance to act on them — for example, if an older version of this extension auto-marked them read, if you accidentally clicked "mark all as read" on iNaturalist, or if you simply want to revisit recent activity. It pulls up to 200 of your most recent viewed updates from the iNaturalist API in a single call and surfaces them in the popup so you can review or re-open them.

Unread + read: combines both — useful if you suspect some items slipped into "read" status but you still want fresh unread items too.

Fetching never marks notifications read on iNaturalist.
Note: Read/recovery fetches use the iNaturalist API and currently include only comments and identifications on observations — not @mentions.

Display

When enabled, clicking a notification opens it in a new tab and removes it from the popup. When disabled, opened notifications stay visible but appear dimmed.

Whitelist — show only matching

When the whitelist toggle is on in the popup, only notifications matching any term (OR logic) are shown.
Matching is case-insensitive. Works on message text, observation ID, or href. Example terms: "rosemary", "added an identification", "188284414" (observation ID).

Blacklist — hide matching

When the blacklist toggle is on in the popup, notifications matching any term are hidden.
Matching is case-insensitive. Works on message text, observation ID, or href.
Whitelist and blacklist can both be used. If both are on, the whitelist first narrows the list, then the blacklist removes unwanted matches.
